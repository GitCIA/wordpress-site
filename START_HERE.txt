================================================================================
                        NEXT STEP: PUSH TO GITHUB
================================================================================

You need to authenticate and push. Choose ONE method:

METHOD 1: SSH Keys (RECOMMENDED - Most Secure)
────────────────────────────────────────────────
1. Generate SSH key:
   ssh-keygen -t ed25519 -C "your-email@example.com"

2. Add public key to GitHub:
   • Go to: https://github.com/settings/keys
   • Click: "New SSH key"
   • Paste content from: ~/.ssh/id_ed25519.pub

3. Update git remote:
   git remote remove origin
   git remote add origin git@github.com:GitCIA/wordpress-site.git

4. Push to GitHub:
   git push -u origin master

METHOD 2: GitHub CLI (EASIEST)
──────────────────────────────
1. Install: https://cli.github.com/

2. Authenticate:
   gh auth login

3. Create repo and push:
   cd d:\staging.cgd\httpdocs
   gh repo create wordpress-site --public --source=. --remote=origin --push

METHOD 3: Manual Web + HTTPS
────────────────────────────
1. Go to: https://github.com/new
2. Create repository: "wordpress-site" (Public)
3. Run these commands:
   git remote remove origin
   git remote add origin https://github.com/GitCIA/wordpress-site.git
   git push -u origin master

================================================================================
                  AFTER GITHUB: SET UP HOSTING (34sp.com)
================================================================================

Once pushed to GitHub, configure your hosting server:

SSH into your hosting:
  ssh user@34sp.com
  cd /path/to/wordpress

Run these commands:
  git init
  git config user.email "admin@34sp.com"
  git config user.name "Hosting Admin"
  git remote add origin git@github.com:GitCIA/wordpress-site.git
  git fetch origin
  git checkout -b master origin/master

Optional: Set up automatic deployment (auto-pull every 15 minutes):
  crontab -e
  # Add: */15 * * * * /path/to/deploy.sh >> /var/log/wordpress-deploy.log 2>&1

================================================================================
                            IMPORTANT REMINDERS
================================================================================

✅ DO:
  • Commit theme and plugin changes
  • Use descriptive commit messages
  • Push regularly to GitHub
  • Keep local and hosting in sync

❌ DON'T:
  • Commit wp-config.php with real credentials
  • Commit /wp/ directory (WordPress core)
  • Commit /wp-content/uploads/ (user media)
  • Commit .env files with passwords
  • Use password for GitHub (use SSH or token)

⚠️  SECURITY:
  • Never commit database credentials
  • Use SSH keys for GitHub instead of passwords
  • Configure wp-config.php per environment (local vs hosting)
  • Keep .env.example but not .env files
